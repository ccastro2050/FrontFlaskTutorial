CUANDO SE MUESTRA: ================== Esta pagina aparece cuando un usuario autenticado intenta acceder a una ruta para la cual NO tiene permiso segun sus roles. Ejemplo: - El usuario "juan@mail.com" tiene rol "Vendedor" - El rol "Vendedor" solo tiene acceso a /producto y /cliente - Juan intenta acceder a /factura (no asignada a su rol) - El middleware (auth_middleware.py) detecta que /factura NO esta en sus rutas_permitidas y muestra esta pagina con codigo HTTP 403. CODIGO HTTP 403: ================ 403 = "Forbidden" (Prohibido). Significa que el servidor entendio el request pero se niega a cumplirlo porque el usuario no tiene permisos. Es diferente de 401 (no autenticado) y 404 (no encontrado). Para que esta pagina NO aparezca, el administrador debe: 1. Ir a "Admin Rutas" y sincronizar las rutas del sistema 2.
